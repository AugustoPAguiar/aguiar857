// OVERVIEW

O módulo Attacking Web Applications with Ffuf no HTB Academy apresenta uma das ferramentas mais eficazes no arsenal de reconhecimento de um pentester. ffuf — Fuzz Faster U Fool — é um web fuzzer rápido e flexível feito para descobrir conteúdo oculto e enumerar superfícies de ataque em aplicações HTTP.

O módulo abrange:

>
[01] Fuzzing de diretórios e páginas com wordlists
[02] Scan recursivo com filtragem de extensões
[03] Enumeração de subdomínios contra alvos reais
[04] VHost fuzzing em domínios roteados internamente
[05] Descoberta de parâmetros GET e POST
[06] Value fuzzing para extrair flags via curl

Este é um walkthrough completo de como trabalhei pelo módulo — os comandos, o raciocínio e as lições que carrego para engagements reais.

::: FINDING HIDDEN DIRECTORIES

A primeira tarefa foi brute-force de diretórios usando o ffuf com a wordlist directory-list-2.3-small.txt contra o alvo. O flag -w especifica a wordlist e -u define a URL alvo, com FUZZ como ponto de injeção:

TERMINAL — BASH

::: EXTENSION FUZZING

Nem todos os ficheiros têm extensão visível. Com o ffuf é possível testar múltiplas extensões (.php, .html, .aspx) usando duas wordlists ao mesmo tempo, com dois keywords distintos:

TERMINAL — BASH
$ ffuf -w /opt/useful/SecLists/Discovery/Web-Content/web-extensions.txt:FUZZ \ -u http://SERVER_IP:PORT/blog/indexFUZZ .php [Status: 200, Size: 0, Words: 1, Lines: 1] .phps [Status: 403, Size: 283, Words: 20, Lines: 10]

// PARTE 2 — SUBDOMAIN ENUMERATION

::: SUBDOMAIN FUZZING

A enumeração de subdomínios testa prefixos contra o domínio alvo. Com o ffuf basta substituir o subdomínio pelo keyword FUZZ na URL:

TERMINAL — BASH
$ ffuf -w /opt/useful/SecLists/Discovery/DNS/subdomains-top1million-5000.txt:FUZZ \ -u https://FUZZ.hackthebox.eu/ www [Status: 200, Size: 22266, Words: 2123, Lines: 251] forum [Status: 200, Size: 75324, Words: 5765, Lines: 489] academy [Status: 302, Size: 0, Words: 1, Lines: 1]

::: VHOST FUZZING

VHost fuzzing difere de subdomain fuzzing porque testa cabeçalhos HTTP Host diretamente contra o IP alvo, permitindo descobrir domínios internos não registados publicamente no DNS:

TERMINAL — BASH
$ ffuf -w /opt/useful/SecLists/Discovery/DNS/subdomains-top1million-5000.txt:FUZZ \ -u http://academy.htb:PORT/ \ -H 'Host: FUZZ.academy.htb' \ -fs 900 admin [Status: 200, Size: 2510, Words: 465, Lines: 72] test [Status: 200, Size: 2501, Words: 462, Lines: 72]
!
DICA: O flag -fs filtra por tamanho de resposta (size). É essencial para remover falsos positivos — respostas que retornam 200 mas têm sempre o mesmo tamanho são páginas de erro genéricas, não resultados reais.
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

// PARTE 3 — PARAMETER DISCOVERY

::: GET PARAMETERS

Descoberta de parâmetros GET ocultos pode revelar funcionalidades não documentadas. O ffuf injeta o FUZZ no lugar do nome do parâmetro na query string:

TERMINAL — BASH
$ ffuf -w /opt/useful/SecLists/Discovery/Web-Content/burp-parameter-names.txt:FUZZ \ -u http://admin.academy.htb:PORT/admin/admin.php?FUZZ=key \ -fs 798 user [Status: 200, Size: 800, Words: 65, Lines: 35]

::: POST PARAMETERS

Para parâmetros POST, o ffuf usa o flag -d para definir o body e -X POST para o método. A wordlist continua a injetar no keyword FUZZ:

TERMINAL — BASH
$ ffuf -w /opt/useful/SecLists/Discovery/Web-Content/burp-parameter-names.txt:FUZZ \ -u http://admin.academy.htb:PORT/admin/admin.php \ -X POST \ -d 'FUZZ=key' \ -H 'Content-Type: application/x-www-form-urlencoded' \ -fs 798 id [Status: 200, Size: 803, Words: 68, Lines: 35]
■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■■

// PARTE 4 — VALUE FUZZING

Com o parâmetro descoberto, o passo final é fuzzing de valores. Geramos uma wordlist de números de 1 a 1000 com bash e testamos cada um como valor do parâmetro id:

TERMINAL — BASH
# Gerar wordlist de IDs numéricos $ for i in $(seq 1 1000); do echo $i; done > ids.txt # Fuzz com POST e filtrar por tamanho $ ffuf -w ids.txt:FUZZ \ -u http://admin.academy.htb:PORT/admin/admin.php \ -X POST \ -d 'id=FUZZ' \ -H 'Content-Type: application/x-www-form-urlencoded' \ -fs 768 73 [Status: 200, Size: 787, Words: 55, Lines: 35] # Extrair flag com curl $ curl -s http://admin.academy.htb:PORT/admin/admin.php \ -X POST -d 'id=73' HTB{p4r4m3t3r_fuzz1ng_1s_k3y!}
